pgSchema Error Codes
====================

NOTE: Most error in pgSchema will be raised by ADO or other libraries.
      The coding in use is a 5432 prefix, followed by 3 digits.

5432001 - This program can only be used with PostgreSQL 7.1 or higher.
5432002 - Domains only exist in PostgreSQL 7.3 or above.
5432003 - NOT NULL can only be set or dropped post column creation in PostgreSQL 7.3 or above.
5432004 - Functions can only be updated in PostgreSQL 7.3 or above.
5432005 - You cannot drop dummy schemas in PostgreSQL versions prior to 7.3.
5432006 - You cannot create schemas in PostgreSQL versions prior to 7.3.
5432007 - You cannot modify dummy schemas in PostgreSQL versions prior to 7.3.
5432008 - Primary Keys can only be added or dropped in PostgreSQL 7.3+
5432009 - Triggers can only be renamed in PostgreSQL 7.3+.
5432010 - User variables can only exist in PostgreSQL 7.3+.
5432011 - Database variables can only exist in PostgreSQL 7.3+.
5432012 - Database security is only available in PostgreSQL 7.3+.
5432013 - Function security is only available in PostgreSQL 7.3+.
5432014 - Language security is only available in PostgreSQL 7.3+.
5432015 - Column statistics are only available in PostgreSQL 7.2+.

Setting up a Development Environment

Before you can work on pgAdmin, you must setup a development environment. It is not practical to develop pgAdmin and have a normally installed version on the same machine because much of the time the components in use in the development system will not be compatible with the installed version. Follow these steps to get started:

  1. Uninstall any installed copies of pgAdmin II.
     
  2. Install WinCVS from http://www.wincvs.org/
     
  3. Select a local cvsroot directory, and set the remote cvsroot as outline here.
     
  4. Checkout copies of the following modules: activex, binaries, pgadmin2, pgschema.
     
  5. Register the .dlls and .ocxs in the binaries directory using regsvr32.exe. pgSchema.dll should be registered first.
     
  6. pgAdmin may be run by just executing pgAdmin2.exe.

If at any time you break compatibility in pgSchema, you may find the whole installation should be recompiled. The BuildAll.bat batch file in the pgadmin2 directory can be used for this.

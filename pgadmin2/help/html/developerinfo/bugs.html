Known Bugs
  • It is not currently possible to drop a database. pgSchema does't close all connections to a database before issuing the 'DROP DATABASE' statement, hence the DROP fails.
     
  • If you attempt to view the data in pg_index or pg_trigger you are likely to get an error like:

    PGADMIN2 caused an invalid page fault in module MSADRH15.DLL at 016f:1f541588.

    As far as I can tell, this is a bug in ADO that occurs in frmSQLOutput:RefreshData at the line

    Select Case rsSQL.Fields(X).Type

    Reasons for this error are completely unknown as it is almost certainly a Microsoft bug.  
     
  • The Publishing Wizard should drop objects in reverse order otherwise it will fail with inherited tables.
     
  • Allow import of null rows in the Migration Wizard.

Known Bugs
  • It is not currently possible to drop a database. pgSchema does't close all connections to a database before issuing the 'DROP DATABASE' statement, hence the DROP fails.
     
  • The data grid sometimes fails to locate rows to update or delete if one or more columns contains a \n. Preliminary investigation indicates that perhaps data in the database has *nix style end of lines which ADO is converting to CrLf. This data cannot then be matched as the source data doesn't contain CrLf...

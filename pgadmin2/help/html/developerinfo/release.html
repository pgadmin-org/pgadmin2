Release Procedure

This document outlines the procedure for producing a pgAdmin II release whether a development or stable version.

  1. Run BuildAll.bat to ensure all binaries are built correctly and all interfaces are compatible.
     
  2. Recompile the HTML Help using the Microsoft HTML Help Workshop.
     
  3. Make sure that CHANGELOG.txt is up to date.
     
  4. Run the Wise for Windows Installer and load installer\pgadmin2.wsi.
     
  5. Update the version number and release comments under General Information and Product Details under the Installation Expert.
     
  6. Click into the Setup Editor and onto the Components tab. Right-click the Components node, and select Rescan Advertising. This will ensure that the Installer registers all COM objects correctly.
     
  7. From the File menu, select Compile to build the MSI file.
     
  8. Test the resulting MSI on as many platforms as possible.

The following steps should be followed for official releases:

  1. Add a CVS tag to mark the release. The format is REL-n_n_n where n_n_n is the full version number (major.minor.build).
     
  2. Zip the MSI file using maximum compression. The zip filename should be pgadmin2-n_n_n.zip where n_n_n is the full version number (major.minor.build).
     
  3. Scan the zip file for viruses using an up-to-date scanner.
     
  4. Upload the file to the download directory on the webserver, and add the details of the download to the downloads table in the database.
     
  5. Update /versions.dat on the webserver.

To Do List

Functionality - These are priority items that are easily possible through SQL.

  • Allow dropping of UNIQUE & PRIMARY contraints by dropping the relevant index.
     
  • Allow the ownership of Sequences, Views & Indexes to be changed.
     
  • Add code to allow pseudo-editing of Views, Triggers & Tables. This would be implemented in pgSchema and would generally involve dropping object and re-creating them with the new changes applied.

Essential Enhancements - These are items that should be considered important, but are not necessarily easy, or even possible.

  • Use cursors/on demand loading in the Datagrid with large recordsets.
     
  • Auto generate rules on views to allow updating of views.
     
  • Add support for Unicode charactersets (UTF-8/16).
     
  • Allow selection of the .mdw file in the Migration Wizard.
     
  • Allow Migration of data only to pre-existing tables in the Migration Wizard.
     
  • Publishing Wizard should remember production databases.

Miscellaneous - These are potentially large projects, that may not be widely used. These are low priority.

  • Add 'Projects' support. Should allow objects to be grouped into projects. A single project should be exportable to an SQL script, or publishable to other databases/servers. Should also include security to restrict access to objects to authorised development team members.
     
  • Build a Log Analyser Plugin to analyse PostgreSQL logs. eg. Watching a user's session, diagnosing errors, possibly generating roll-back data, watching for transaction processing "hot-spots" during a day or week, etc.
     
  • Build an entity relationship diagramming plugin. This should reverse engineer the database to build a user updateable ER diagram of a database.
     
  • Add a Plugin to drive erServer.
     
  • Write PostgreSQL Installer for Cygwin PostgreSQL. This should run from pgAdmin and auto download & install Cygipc, a minimal Cygwin and PostgreSQL.
     
  • Add statistics reports for 7.2. Output should be HTML(?) - Crystal Reports should *not* be used.
